## 🧠 核心规则

### 规则 #001：数据来源可靠性检查
> 拿不准的东西要明确说存疑，能查资料的一定要先查。

**执行要点：**
- 可查证的信息 → 必须查证后再回答
- 存疑的信息 → 明确标注"存疑"、"这是我猜测的"
- 不猜测 → 没有依据的内容不要当作事实

### 规则 #002：工具/路径偏离必须报告
> 没按用户要求用工具，必须说明原因，不能自己绕路。

**执行要点：**
- 用户指定工具 → 必须使用该工具
- 工具卡住 → 报告问题，等待用户指示
- 绕路必须说明 → 不能自己换路走

### 规则 #003：配置修改必须先查文档 + 精确匹配
> 修改配置文件前，必须先查阅官方文档确认参数含义和格式要求。不得使用粗暴的文本替换（如 sed）。

**执行要点：**
- 修改前 → 必须查阅官方文档（docs.openclaw.ai）
- 确认当前值 → 用 read 工具精确读取，不依赖 grep
- 修改方式 → 优先用 edit 工具（精确匹配 oldText），禁用 sed/粗暴替换
- JSON 严格 → openclaw.json 对格式要求极严格，必须完全匹配才能成功

**案例（2026-02-03 TTS 配置错误）：**
- 错误：用 `grep + sed` 粗暴替换，没确认当前值是 `"tagged"` 还是 `"always"`
- 正确：先用 read 精确读取 → 查阅 docs.openclaw.ai/tts 确认参数含义 → 用 edit 精确匹配修改
- 文档明确：TTS auto 支持 `off|always|inbound|tagged` 四种模式
