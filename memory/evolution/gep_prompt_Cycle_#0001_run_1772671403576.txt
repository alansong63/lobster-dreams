### ⚠️ 重要规则
- **必须使用 wrapper 脚本**，禁止直接调用 `todo.sh`
- wrapper 会自动设置 `TODO_DB` 环境变量，避免数据库创建在错误位置
- 直接调用 `todo.sh` 且未设置 `TODO_DB` 时，会报错阻止

### 核心功能
| 功能 | 命令 |
|------|------|
| 添加任务 | `bash ~/.openclaw/workspace/skills/todo-management/scripts/todo-wrapper.sh entry create "任务" --due="tomorrow"` |
| 查看任务 | `bash ~/.openclaw/workspace/skills/todo-management/scripts/todo-wrapper.sh entry list` |
| 提醒检查 | `bash ~/.openclaw/workspace/skills/todo-management/scripts/todo-wrapper.sh reminder check --days=3` |

### 分组结构（按角色）
| 分组 | 用途 | 负责人 |
|------|------|--------|
| Inbox | 收件箱/未分类 | - |
| director | Lobster_Bro（主 AI）的任务 | Lobster_Bro |
| assistant | Alan 的工作和生活提醒 | 搭搭（半生产环境） |
| manager | 主任的任务 | 主任 |
| clerk | 文清的任务 | 文清 |

### 搭搭使用规则
- 搭搭负责维护 Alan 的工作和生活提醒（半生产环境）
- **所有 Alan 的任务 → 放入 `assistant` 分组**
- 搭搭调用示例：
  ```bash
  bash .../todo-wrapper.sh entry create "流量卡到期充值提醒" --due="2026-03-25" --group="assistant"
  ```

### 提醒机制
- `reminder check` 返回 JSON，包含 overdue/today/upcoming 三个数组
- 集成脚本：`scripts/todo-reminders-integration.sh`（格式化输出，调用前需设置 `TODO_DB`）
- 建议集成到 HEARTBEAT.md 或 cron

|----------------|--------|
| 飞书 Bitable | 本地 SQLite |
| 无提醒功能 | 完整提醒系统 |
| 无截止日期 | 支持 due_date |
| 文档缺失 | 完整 SKILL.md |
