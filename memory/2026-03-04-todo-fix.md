# 2026-03-04 TODO 数据库路径问题修复

## 问题回顾

搭搭在使用 TODO 系统时，数据库被创建在错误位置：
- **正确位置**: `~/.openclaw/workspace/todo.db`
- **错误位置**: `~/.openclaw/workspace/skills/todo-management/todo.db`

导致数据分散在两个数据库中。

---

## 根因分析（5 个深层次问题）

### 1. `todo.sh` 缺少防御性检查
```bash
DB="${TODO_DB:-$(pwd)/todo.db}"
```
未设置 `TODO_DB` 时，数据库会创建在当前工作目录。

### 2. `todo-reminders-integration.sh` 有同样漏洞
```bash
TODO_DB="${TODO_DB:-$(pwd)/todo.db}"
```

### 3. MEMORY.md 文档指向错误
文档指向 `todo.sh` 而不是 `todo-wrapper.sh`，误导用户直接调用。

### 4. `todo-alias.sh` 设计有隐患
先 `cd` 进 skill 目录再调用 `todo.sh`，如果 export 失效就会出错。

### 5. SKILL.md 警告不够醒目
警告放在大段文字里，容易被忽略。

---

## 修复内容

### ✅ P0: 修改 `todo.sh`
添加防御性检查，检测到在 skill 目录运行且未设置 `TODO_DB` 时主动报错：
```bash
error: 检测到在 skill 目录内运行，且未设置 TODO_DB 环境变量
请使用以下方式之一：
  1. 使用 wrapper: bash .../todo-wrapper.sh ...
  2. 设置环境变量：export TODO_DB=~/.openclaw/workspace/todo.db
```

### ✅ P0: 修改 `todo-reminders-integration.sh`
添加同样的防御性检查。

### ✅ P1: 更新 MEMORY.md
- 修正脚本路径指向 `todo-wrapper.sh`
- 添加"⚠️ 重要规则"强调必须使用 wrapper
- 更新命令示例使用完整路径

### ✅ P1: 更新 SKILL.md
- 将警告移到文件最开头，添加"必读"标识
- 用 ✅/❌ 清晰标注正确和错误用法
- 更新 CLI 调用说明

### ✅ P2: 添加 bash alias
在 `~/.bashrc` 末尾添加：
```bash
alias todo='bash ~/.openclaw/workspace/skills/todo-management/scripts/todo-wrapper.sh'
```
