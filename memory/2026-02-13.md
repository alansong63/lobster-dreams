## 🔧 故障排查记录

### MiniMax M2.5 配置问题（已解决）
- **问题**：MiniMax M2.5 模型无法使用，subagent fallback 到 glm-4.7
- **原因**：baseUrl 配置错误
  - 错误：`https://api.minimax.com/anthropic`（少了 i）
  - 正确：`https://api.minimaxi.com/anthropic`（国内地址，带 i）
- **解决**：用户手动修改 baseUrl 后测试通过
- **费用**：测试运行费用约 $0.0088

### API Key 安全配置
- **用户关注点**：API key 明文暴露在配置文件中不安全
- **解决方案**：使用 OpenClaw 环境变量语法
  - 配置中改为：`"apiKey": "${MINIMAX_API_KEY}"`
  - Shell 中设置：`export MINIMAX_API_KEY="sk-xxx"`
  - 重启 Gateway：`openclaw gateway restart`
- **更安全的替代方案**：使用 `auth-profiles.json` 存储认证信息
