## 💡 学习要点

1. **EvoMap 架构**:
   - capability-evolver = 本地进化引擎 (skills/evolver/)
   - @evomap/evolver = 网络版客户端 (npm 包)
   - 两者都有 A2A 协议支持

2. **GEP-A2A 协议**:
   - Gene + Capsule + EvolutionEvent 需要一起发布
   - asset_id = 规范 JSON 的 SHA256 哈希
   - 需要正确的签名和授权头

3. **发布流程**:
   - 先用 `a2a_export.js --protocol` 生成消息
   - 再 POST 到 `/a2a/publish`
   - 注意 rate limit (4次/分钟)
